# Material-Aware Scoring Framework — Reference Spec

> **Method:** Score each section 0–10 using the band rubric below, then apply
> the "start at 10, subtract for issues" checklist to land the final number.
> Aggregate via weighted overall score; weights shift by audit mode.

---

## Per-Section Rubrics

Each section rubric contains:
1. **Band table** — criteria for each score range
2. **Deduction checklist** — start at 10, subtract for each issue; bonuses where noted

---

### 📷 Photo (0–10)

> ⚠️ **Assessability note:** Score only if an image is provided in the audit material.
> LinkedIn PDF exports do not include the profile photo. If no image is provided,
> mark as **"not assessable from PDF"** and flag it as a recommended upload/review item.

**Band table:**

| Score | Criteria |
|-------|----------|
| 9–10 | Professional headshot, excellent lighting, face fills 60–70% of frame, warm expression, clean/neutral background, industry-appropriate attire |
| 7–8 | Good quality, professional appearance, decent lighting, appropriate background, face clearly visible |
| 5–6 | Acceptable photo, some lighting/background/cropping issues, face visible but not optimised |
| 3–4 | Low quality, unprofessional setting, distracting background, or poorly cropped |
| 1–2 | Very low quality, inappropriate context, barely visible face, or obviously a non-headshot repurposed |
| 0 | No photo uploaded, or default LinkedIn avatar |

**Deduction checklist (start at 10):**

| Issue | Deduct |
|-------|--------|
| Face not clearly visible | −3 |
| Sunglasses or hat obscuring face | −2 |
| Cropped from a group photo | −2 |
| Distracting, cluttered, or wildly off-brand background | −1 |
| Visibly outdated (estimated >5 years old) | −1 |
| Poor or harsh lighting | −1 |
| Low resolution or blurry | −1 |
| Unprofessional attire for industry | −1 |
| Not smiling / closed or harsh expression | −0.5 |
| Face fills <60% of frame | −0.5 |

**Bonuses:**

| Bonus | Add |
|-------|-----|
| Verified badge visible in frame or well-styled | +0.5 |
| Studio-quality lighting and composition | +0.5 |
| Perfect industry-context alignment (e.g., lab coat for researcher, formal for finance) | +0.5 |

### 🖼️ Banner / Cover Image (0–10)

> ⚠️ **Assessability note:** Score only if a banner image is provided in the audit material.
> LinkedIn PDF exports do not include the banner. Mark as **"not assessable from PDF"** if absent;
> flag as a high-impact quick win (the default blue/grey banner is a missed branding opportunity).

**Band table:**

| Score | Criteria |
|-------|----------|
| 9–10 | Custom, on-brand banner that reinforces value prop; readable text (if any); visually consistent with headline and role; professional design quality |
| 7–8 | Custom banner, mostly on-brand, minor design quality issues or slightly off-message |
| 5–6 | Custom banner but generic (stock photo with no message), low effort, or poor text legibility |
| 3–4 | Barely custom — stretched/pixelated image, obvious template misuse, or contradicts professional positioning |
| 1–2 | Clearly amateurish, inappropriate imagery, or severely distorted |
| 0 | Default LinkedIn banner (blue/grey), or no banner set |

**Deduction checklist (start at 10):**

| Issue | Deduct |
|-------|--------|
| Default LinkedIn banner | −10 (floor at 0) |
| No connection to professional identity/value prop | −3 |
| Text illegible (too small, poor contrast) | −2 |
| Stretched, pixelated, or distorted | −2 |
| Generic stock photo with no personalisation | −1.5 |
| Misaligned with target role/industry | −1 |
| Outdated company logo or old role references | −1 |

**Bonuses:**

| Bonus | Add |
|-------|-----|
| Reinforces a specific value prop or call-to-action | +0.5 |
| Includes social proof (book cover, stage photo, brand logos) | +0.5 |

### 📝 Headline (0–10)

> Always scoreable from text material.

**Band table:**

| Score | Criteria |
|-------|----------|
| 9–10 | Specific value prop + target keywords + differentiation; clearly answers "what do I do and who do I help"; naturally includes 2–4 high-value keywords; compelling and skimmable |
| 7–8 | Good specificity and most relevant keywords present; minor missed opportunities (vague qualifier, one keyword gap) |
| 5–6 | Job title present but limited differentiation; some keywords but generic phrasing; would not stand out in search results |
| 3–4 | Only title + company name; no value prop; minimal keyword signal |
| 1–2 | Vague, generic, or actively off-putting; "Seeking opportunities," "Open to work," or empty phrases that convey nothing |
| 0 | Blank, default ("LinkedIn Member"), or missing entirely |

**Deduction checklist (start at 10):**

| Issue | Deduct |
|-------|--------|
| No target keywords in headline | −3 |
| No value prop (just title + company) | −2 |
| "Seeking opportunities" or "Open to work" as the full/primary headline | −2 |
| Vague buzzwords with no specifics ("passionate," "results-driven") | −1.5 |
| Exceeds 220 characters (truncates in search results) | −1 |
| No differentiation from thousands of others with same title | −1 |
| Missing who-you-help or impact framing | −1 |

**Bonuses:**

| Bonus | Add |
|-------|-----|
| Includes specific metric or credential as proof | +0.5 |
| Compelling hook word at start (not just title) | +0.5 |

### 📄 About / Summary (0–10)

> Always scoreable from text material.

**Band table:**

| Score | Criteria |
|-------|----------|
| 9–10 | Compelling opening hook (first 3 lines visible before "see more"); clear value prop; 3–5 key achievements with specifics/metrics; keywords woven in naturally; ends with a call-to-action; 1,500–2,000 chars (≈250–340 words); first-person voice |
| 7–8 | Good hook and value prop; some quantified achievements; most keywords present; CTA present but weak or missing; readable and professional |
| 5–6 | Decent summary but starts with name/title (not a hook); achievements present but vague; missing CTA; or slightly too short/long |
| 3–4 | Generic filler ("I am a dedicated professional…"); no hook; no specifics; no CTA; or mostly a resume paste |
| 1–2 | Near-empty, placeholder text, third-person robot-speak, or incoherent |
| 0 | Blank About section |

**Deduction checklist (start at 10):**

| Issue | Deduct |
|-------|--------|
| No hook — starts with name, title, or "I am" | −2 |
| Wall-of-text (no line breaks, hard to skim on mobile) | −2 |
| No quantified achievements (everything vague) | −2 |
| Missing call-to-action at the end | −1.5 |
| No target keywords present | −1.5 |
| Written in third person | −1 |
| Under ~600 chars (≈100 words) — wasted prime real estate | −1 |
| Over ~2,600 chars (≈430 words) — buries key points; exceeds LinkedIn's hard cap | −0.5 |

**Bonuses:**

| Bonus | Add |
|-------|-----|
| Specific, memorable opening line that creates curiosity | +0.5 |
| Social proof embedded naturally (client, publication, award) | +0.5 |

### 💼 Experience (0–10)

> Always scoreable from text material.

**Band table:**

| Score | Criteria |
|-------|----------|
| 9–10 | Every role has 3–5 bullet points; bullets lead with strong action verbs; ≥50% of bullets are quantified; clearly shows progression/growth; keywords naturally embedded; dates and titles complete |
| 7–8 | Most roles have bullets; mix of quantified and qualitative; keywords mostly present; minor gaps (one role thin, or some vague bullets) |
| 5–6 | Some roles have bullets, others are bare; quantification patchy; progression unclear; important keywords missing from descriptions |
| 3–4 | Most roles have no description or only 1-line duty statements; no quantification; reads like a form-filled resume dump |
| 1–2 | Experience section nearly empty; date gaps unexplained; titles without descriptions; or only current role listed |
| 0 | No experience section filled in |

**Deduction checklist (start at 10):**

| Issue | Deduct |
|-------|--------|
| Roles with no description (just title + dates) | −1.5 per role (cap −4) |
| Zero quantified results across entire section | −3 |
| No action verbs — passive or duty-list language | −2 |
| Target keywords absent from experience descriptions | −1.5 |
| Unexplained gap >6 months | −1 |
| No evidence of career progression or increasing responsibility | −1 |
| Inconsistent date formats or missing months | −0.5 |

**Bonuses:**

| Bonus | Add |
|-------|-----|
| Compelling "headline" for a role beyond just title | +0.5 |
| Media, links, or publications attached to a role | +0.5 |

### 🛠️ Skills (0–10)

> Always scoreable from text material.

**Band table:**

| Score | Criteria |
|-------|----------|
| 9–10 | 25–50 skills listed; top 3 pinned skills are the highest-value keywords for the target role; mix of hard skills, tools, and soft skills; all reflect actual experience; endorsements visible for top skills |
| 7–8 | 15–25 skills; mostly relevant; top 3 are good but not perfectly optimised; minor gaps in target keywords |
| 5–6 | 5–15 skills; some relevant but missing key terms; top 3 not strategically chosen; no obvious endorsements |
| 3–4 | Fewer than 5 skills or clearly incomplete; major keyword gaps for target role |
| 1–2 | 1–3 generic skills (e.g., "Microsoft Office" only) with nothing role-specific |
| 0 | Skills section empty or not filled in |

**Deduction checklist (start at 10):**

| Issue | Deduct |
|-------|--------|
| Fewer than 10 skills listed | −3 |
| Top 3 pinned skills are not the most searchable/valuable for target role | −2 |
| Missing primary target-role keywords as skills | −2 |
| Stale/irrelevant skills dominate the list | −1 |
| No endorsements visible on top skills | −1 |
| Over-indexed on soft skills, under-indexed on technical/tool skills | −0.5 |

**Bonuses:**

| Bonus | Add |
|-------|-----|
| Skill assessments passed (LinkedIn badges) on top skills | +0.5 |
| Top 3 skills match exactly the most-searched terms in target role | +0.5 |

### ⭐ Recommendations (0–10)

> Always scoreable from text material (recommendation text is included in LinkedIn PDF exports).

**Band table:**

| Score | Criteria |
|-------|----------|
| 9–10 | 5+ recommendations; variety of recommenders (peer, manager, direct report, client); each recommendation is specific with named achievements; recommendations align with the target role narrative |
| 7–8 | 3–4 recommendations; mostly specific; reasonable recommender variety; minor narrative gaps |
| 5–6 | 2–3 recommendations; some specific, some generic ("great to work with"); limited variety |
| 3–4 | 1–2 recommendations; very generic ("hard worker," "great person"); or from only one type of relationship |
| 1–2 | 1 extremely generic recommendation that adds no proof value |
| 0 | No recommendations |

**Deduction checklist (start at 10):**

| Issue | Deduct |
|-------|--------|
| Fewer than 3 recommendations | −3 |
| All recommendations from peers only (no manager or client voice) | −2 |
| All recommendations are generic (no specifics, achievements, or results) | −2 |
| Recommendations contradict or are irrelevant to target role | −1.5 |
| Most recommendations are 5+ years old | −1 |
| You have not given any recommendations (reduces reciprocity signals) | −0.5 |

### 📊 Activity / Content (0–10)

> ⚠️ **Assessability note:** Activity metrics (post frequency, engagement rate, impressions)
> require access to LinkedIn Creator analytics and are NOT available in a PDF export.
> Score only if the user provides activity data directly. Otherwise mark as
> **"not assessable from PDF"** and give target benchmarks for the user to self-check.

**Band table:**

| Score | Criteria |
|-------|----------|
| 9–10 | Posts 3–5× per week; content consistently on-topic for target role/industry; high engagement (3%+ engagement rate); content showcases expertise with original insights; mix of formats (text, documents, video) |
| 7–8 | Posts 1–2× per week; mostly on-topic; decent engagement; some original content alongside reshares |
| 5–6 | Posts a few times per month; mixed topics; engagement exists but low; primarily reshares without added commentary |
| 3–4 | Posts rarely (once a month or less); minimal engagement; mostly off-topic |
| 1–2 | Almost never posts; virtually no engagement visible; activity section shows old or irrelevant content |
| 0 | No activity visible or content entirely off-brand/damaging |

**Deduction checklist (start at 10):**

| Issue | Deduct |
|-------|--------|
| Posts less than once per month | −4 |
| Content is entirely off-topic for target role | −3 |
| Engagement rate <1% (for Creator-mode or brand goals) | −2 |
| No original content — only reshares with no commentary | −1.5 |
| Last post is >3 months old | −1.5 |
| Controversial or unprofessional posts visible | −2 (situational) |

**Bonuses:**

| Bonus | Add |
|-------|-----|
| "Creator mode" enabled with featured content | +0.5 |
| Newsletter or LinkedIn article series active | +0.5 |

### "Not Assessable from PDF" Weight Redistribution

When Photo, Banner, or Activity are marked "not assessable from PDF," redistribute their weight proportionally to the remaining assessable sections rather than scoring a zero (which would unfairly penalise the user for data not available in the export).

**Rule:** Sum the weights of non-assessable sections → distribute that total proportionally across all assessable sections based on their relative weights.

**Example:** In `job` mode, if Photo (6%) and Banner (4%) are both not assessable:
- Remaining assessable weight = 90% across 6 sections
- Each assessable section's adjusted weight = original_weight / 90 × 100
- Headline: 22 → 24.4%; Experience: 22 → 24.4%; Skills: 16 → 17.8%; About: 14 → 15.6%; Recommendations: 10 → 11.1%; Activity: 6 → 6.7%

Always note in the audit output which sections were excluded and why, so the user understands the score reflects available material only.

## Recruiter 6-Second Scan Rule

### What It Is

> **Canonical definition:** the 6-second scan is defined in full in
> `industry-best-practices.md §6` (surfaces, the CLICK/SKIP test, and the complete
> trigger checklists). That file is the single source of truth. If anything here
> appears to conflict with §6, **§6 wins.** This section keeps only the output
> format the audit emits inline.

In one line: before per-section scoring, simulate what the target audience (per `mode`
and `target_audience` from intake) sees in the first 6 seconds across the three surfaces
named in §6 — headline, first ~300 chars of About, and current role — and decide CLICK
or SKIP using the trigger checklists in §6.2.

---

### 6-Second Scan Output Format

Report this as a mini-section at the top of the audit, before section scores:

```
🔍 6-Second Scan (recruiter / target audience POV)

Headline:  "Senior PM | B2B SaaS | Roadmaps that drive growth"
About opener:  "I've launched 12 products that collectively drove $40M in ARR …"
Current role:  Senior Product Manager at Acme Inc

Verdict: ✅ CLICK  (specific value prop, keywords up front, quantified hook)

Issues flagged:
• None critical — headline is strong

OR

Verdict: ⚠️ SKIP  (reason: vague headline, no value prop, wall-of-text About)

Issues flagged:
• Headline contains no target keywords → likely won't surface for "Senior PM" searches
• About opener is a wall of text → mobile readers bounce immediately
```
